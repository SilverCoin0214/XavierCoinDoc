
# 1. 工厂模式

工厂模式其实就是将创建对象的过程单独封装， 抽取其中共性的东西。
有构造函数的地方，就应该想到工厂模式。

本质的理解就在于， 只需要提供相应的参数，获得产出。 中间封装的过程不需要了解，所有产出的结果都有相同的属性。

---

开放封闭原则： 对扩展开放， 对修改封闭。  软件实体可以拓展，但不可以修改。

---

### 抽象工厂模式

- 抽象工厂（抽象类， 不能用于被生成实体）：用于声明最终目标产品的共性。抽象工厂可以有多个，每一个抽象工厂对应着一类产品，成为产品族。
- 具体工厂（用于生产产品族里的一个具体产品）：继承自抽象工厂，实现了抽象工厂里声明的那些方法，用于创建具体的产品的类。
- 抽象产品（抽象类， 它不能被用于生成具体实例）：具体产品类的共性各自抽离，便对应到了各自的抽象产品类。
- 具体产品（用于生成产品族里的一个具体产品所依赖的更细粒度的产品）

# 2. 单例模式

保证一个类仅有一个实例， 并提供一个访问它的全局访问点， 这样的模式叫做单例模式。

单例的思想在于不论通过多少次创建， 最终返回的都是第一次所创建的实例。 因而内部一定存在一个判断就是该实例是否已经存在，存在就直接返回该实例，不存在就创建。

实现一个storage

```js
class Storage{
  static getInstance() {
	  if (!Storage.instance) {
		  Storage.instance = new Storage()
	  }
	  return Storage.instance
  }

  getItem(key) {
    return LocalStorage.getItem(key)
  }

  setItem(key, value) {
    return LocalStorage.setItem(key, value)
  }
}

```

## 3. 原型模式

原型模式本身就是一种编程范式，对于JS来说，原型模式就是面向对象的根基。 因为要创建对象时，本身就会找一个对象作为原型，然后通过克隆原型的方式创建出一个和原型一样的对象。
